Документ предназначен для ввода остатков на предприятиях, где уже велся бухгалтерский учет
вручную, либо для новых предприятий (только для формиро-вания размера уставного фонда).   
Ввод остатков производится на конец предыдущего отчетного периода (год, квартал, месяц).
Так для перехода на автоматизированное ведение бухгалтерского учета начиная с первого квартала
текущего года остатки вносятся на 31 декабря предыдущего года.
Особенностью документа является использование вспомогательного счета "00", который не влияет
ни на какие отчетные результаты и служит только для ввода остатков. Поскольку программа 
оперирует проводками (обязательно наличие дебета и кредита), то все остатки вносятся в
корреспонденции с нулевым счетом.Так активные счета будут корреспондировать со счетом "00"
по кредиту, а пассивные по дебету. В результате равенства актива и пассива баланса обороты
по дебету и кредиту счета "00" будут одинаковы и счет будет закрыт. Отсутствие сальдо по счету
"00" является подтверждением правильности переноса бухгал-терского баланса в программу.
Последовательность заполнения документа:
1. Проставляем дату документа (изначально датой документа является рабочая дата);
2. Выбираем валюту, если вводятся остатки по счетам с многовалютным учетом. По умолчанию
этот реквизит содержит значение "молдавский лей";
2. Выбираем счет по которому вводятся остатки. После выбора счета программа уведомит вас о
том, куда будут внесены остатки (дебет или кредит);
3. В табличной части вносим аналитики по выбранному счету из соответствующих справочников
к которым привязан счет остатков. Узнать о взаимосвязи счетов и аналитики (субконто) по ним 
можно просмотрев "План бухгалтерских счетов по НСБУ" находящийся в программе; 
4. Если по счету ведется количественный учет - вводим количественное вы-ражение остатков;
5. Если по счету ведется валютный учет - вводим сумму валюты;
6. Последней ячейкой к заполнению является ячейка для ввода суммы остатка в леях;
7. Нажатие клавиши Enter является подтверждением правильности вноса информации по строке;
8. При окончании заполнения строки программа автоматически предложит заполнение следующей
строки путем открытия соответствующего справочника по данному счету для выбора новой
аналитики.При наличии дополнительных позиций ввод  можно продолжить, при их отсутствии
нажать на кнопку "Закрыть" в нижней части экрана, либо клавиши "Esc" - справочник будет 
закрыт и вы вернетесь в документ.После чего надо будет удалить зарезервированную
дополнительную строчку в документе содержащую текущий номер строки без какой-либо
дополнительной информации. Это осуществляется нажатием клавиши "Esc".
ВНИМАНИЕ:
1. На каждый счет и субсчет формируется свой, ОТДЕЛЬНЫЙ документ по вводу остатков.
Исправление счета остатков в шапке документа при введенной аналитике не допускается. 
2. Ввод минусовых сумм в документе не допускается в соответствии со стандартами НСБУ
(активный счет не может иметь остаток по кредиту и наоборот, пассивный - остаток по дебету).
Печать:
Получение печатной копии документа осуществляется нажатием на кнопку "Печать". 
Запись и проведение документа:
Сохраняем документ путем нажатия на кнопку "ОК". На вопрос "Сохранить документ?" отвечаем
"Да" для записи его в базу и "Нет" для отказа. При положительном ответе появится вопрос 
"Провести документ?" - ответ "Да" сформирует по документу проводки, ответ "Нет" - проводки
формироваться не будут, но документ будет сохранен в базе и при необходимости к нему можно
будет вернуться.
Кнопка "Закрыть" предложит вам сохранить документ без формирования проводок по нему.

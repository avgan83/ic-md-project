Документ "Начисления" служит для ввода сумм по видам начислений и удержании в разрезе 
сотрудников.
Порядок начисления заработной платы и списание ее на расходы отчетного периода регулируется
положениями НСБУ 2 "Товарно-материальные запасы", НСБУ 3 "Состав затрат и расходов
предприятия", НСБУ 11 "Строительные подряды", НСБУ 13 "Учет нематериальных активов",
НСБУ 16 "Учет долгосрочных материальных активов", Налоговым кодексом РМ.
Последовательность заполнение документа:
1. Проставляем дату документа (изначально датой документа является рабочая дата);
2. Проставляем количество рабочих дней в месяце. Изначально эта ячейка имеет значение
установленное в константах. Данные о количестве рабочих дней в месяце необходимы для
пересчета начислений по сотрудникам в случае не полностью отработанных рабочих дней в месяце;
3. Выбираем счет затрат зарплаты (по умолчанию - 713.3) и аналитику по нему из справочника
"Затраты";
4. Выбираем счет затрат зарплаты социального фонда (по умолчанию - 713.31) и аналитику по нему
из справочника "Затраты";
5. Выбираем счет больничных листов (по умолчанию - 533.1) и аналитику по нему из справочника 
"Затраты";
6. В поле "Вид деятельности" указывается вид деятельности с которым связаны данные начисления
заработной платы. Это позволит в конце любого отчетного периода получить информацию о доходах
и расходах в разрезе видов деятельности предприятия;
7. В ячейку "Сотрудник" вносится сотрудник по которому будет осуществлено начисление 
заработной платы;
8. Ячейке "Оклад" автоматически появится сумма оклада который сотрудник имеет на дату
создания документа "Начисление заработной платы" из соответствующей истории.
ВНИМАНИЕ:
Изменение суммы оклада по сотруднику невозможно из данного документа. Корректировка 
суммы оклада доступна только в справочнике "Сотрудники";
9. В ячейку "Дни" вносится количество рабочих дней фактически отработанных сотрудником;
10. Ячейка "Сумма" отображает пересчет оклада по фактически отработанным сотрудником дней
по отношению к количеству рабочих дней в месяце;
11. Далее заполняются другие возможные виды начислений, такие как: премия, больничный лист,
отпускные, аренда.
ВНИМАНИЕ:
Данные значения рассчитываются вручную и требуют ручного ввода конечных цифр в
соответствующие ячейки;
12. Ячейка "Итого" покажет общую сумму начисления по конкретному со-труднику;
13. Следующие две колонки предназначены для удержания алиментов и прочих удержание.
Порядок заполнения идентичен: вводится счет алиментов или прочих удержаний, вводится
аналитика по ним и сумма удержаний;
14. Нажатие клавиши Enter является подтверждением правильности вноса информации по строке;
15. При окончании заполнения строки программа автоматически предложит заполнение
следующей строки путем открытия справочника "Сотрудники" для выбора новой аналитики. 
При наличии дополнительных позиций ввод  можно продолжить, при их отсутствии нажать на 
кнопку "Закрыть" в нижней части экрана, либо клавиши "Esc" - справочник будет закрыт и вы
вернетесь в документ. После чего надо будет удалить зарезервированную дополнительную
строчку в документе содержащую текущий номер строки без какой-либо дополнительной
информации. Это осуществляется нажатием клавиши "Esc".
Запись и проведение документа:
Сохраняем документ путем нажатия на кнопку "ОК". На вопрос "Сохранить документ?" отвечаем
"Да" для записи его в базу и "Нет" для отказа. При положительном ответе появится вопрос 
"Провести документ?" - ответ "Да" сформирует по документу проводки, ответ "Нет" - проводки
формироваться не будут, но документ будет сохранен в базе и при необходимости к нему можно
будет вернуться.
Кнопка "Закрыть" предложит вам сохранить документ без формирования проводок по нему.
